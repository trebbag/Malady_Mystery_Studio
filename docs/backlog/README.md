# Backlog usage

The backlog is written in a Codex-friendly way:
- one ticket per file,
- explicit dependencies,
- explicit acceptance criteria,
- explicit out-of-scope notes.

## Suggested rule

Do not let Codex implement a ticket unless its dependencies are already complete or intentionally stubbed.

## Suggested team workflow

1. promote a ticket from Proposed to Ready,
2. link it to a branch,
3. implement,
4. review against acceptance criteria,
5. mark Done or return with follow-up tickets.
